Git

Git en este caso lo estoy incluyendo a este proyecto, es un sistema de control de versiones distribuido que permite a los desarrolladores gestionar y realizar un seguimiento de los cambios en su código fuente. Es una herramienta fundamental en el desarrollo de software moderno, ya que facilita la colaboración entre equipos y la gestión eficiente de proyectos.


  • Características de Git:
    • Control de versiones: Permite realizar un seguimiento de los cambios en el código a lo largo del tiempo.
    • Ramas: Facilita la creación de ramas para trabajar en nuevas características sin afectar la versión principal.
    • Fusión: Permite combinar cambios de diferentes ramas de manera eficiente.
    • Colaboración: Facilita el trabajo en equipo al permitir que varios desarrolladores trabajen en el mismo proyecto simultáneamente.
  • Iniciar un nuevo proyecto
    • git config --global user.name "Tu Nombre": Configura tu nombre de usuario para las confirmaciones.En la carpeta de tu proyecto
    • mkdir mi-primer-proyecto
    • cd mi-primer-proyecto
    • touch index.html README.md: crea un nuevo fichero al proyecto
    • git init: Inicializa un nuevo repositorio Git en el directorio actual.
    • git add Index.html y más: Agrega archivos al área de preparación para su confirmación.
    • git commit -m "Primer commit: estructura inicial": Guarda los cambios en el repositorio local.
    • git checkout -b develop: Crea una ramma llamada develop(copia exacta de main), y ahora todos los cambios seran en develop, tu nueva rama
    • git branch: Muestra las ramas existentes
    • git add index.html: esto hara que se confirme los cambios que hiciste en la rama y siempre acompañado de commit
    • git commit -m "Agregando cambios a la rama develop": Guarda los cambios en la rama actual.
    • Se abrira README.md desde el editor(vs code) y documenta los cambios. Mismo procedimiento para guardar los cambios add y commit
    • git add README.md: Confirmar cambios
    • git commit -m "Agregando cambios a la rama develop": Guarda los cambios en la rama actual.
    • git checkout main: Cambia a la rama principal (main). Si ya no tienes nada que cambiar.
    • git merge: Combina cambios de diferentes ramas. Asegurate de confrimar los cambios ya que se fusionara la rama a tu main
    • git push: Envía los cambios al repositorio remoto.
    • ls: verificar que todo este correctamente en el archivo del proyecto
    • git status: Verifica el estado de los archivos y cambios en el repositorio.
    • git pull: Descarga y fusiona cambios desde el repositorio remoto.
    • git clone: Crea una copia local de un repositorio remoto.
    • git status: Muestra el estado actual del repositorio y los archivos modificados.
    • git log: Muestra el historial de confirmaciones del repositorio.
  • Más Comandos de Git
    • git stash: Guarda temporalmente los cambios no confirmados para limpiar el área de trabajo.
    • git remote: Gestiona los repositorios remotos asociados al repositorio local.
    • git fetch: Descarga cambios desde el repositorio remoto sin fusionarlos.
    • git reset: Deshace cambios en el área de preparación o en el repositorio local.
    • git tag: Marca un punto específico en la historia del repositorio.
    • git diff: Muestra las diferencias entre archivos o confirmaciones.
    • git cherry-pick: Aplica cambios de una confirmación específica a la rama actual.
    • git rebase: Reaplica cambios de una rama sobre otra, manteniendo un historial lineal.
    • git bisect: Ayuda a encontrar un error en el historial de confirmaciones mediante búsqueda binaria.
    • git blame: Muestra quién modificó cada línea de un archivo y cuándo.
    • git reflog: Muestra un registro de todas las acciones realizadas en el repositorio.
    • git config: Configura opciones de Git, como el nombre de usuario y el correo electrónico.
    • git archive: Crea un archivo comprimido de un repositorio o una parte de él.
    • git clean: Elimina archivos no rastreados del directorio de trabajo.
    • git submodule: Permite gestionar repositorios dentro de otros repositorios.
    • git worktree: Permite trabajar en múltiples ramas simultáneamente en diferentes directorios.
    • git grep: Busca texto en el contenido de los archivos del repositorio.
    • git shortlog: Resume el historial de confirmaciones por autor.
    • git show: Muestra información detallada sobre una confirmación específica.
    • git reflog expire: Elimina entradas antiguas del registro de referencias.
    • git gc: Limpia y optimiza el repositorio eliminando archivos innecesarios.
    • git notes: Permite agregar notas a las confirmaciones sin modificar el contenido.
    • git rerere: Ayuda a resolver conflictos de fusión automáticamente.
    • git worktree prune: Elimina referencias de trabajo no utilizadas.
    • git bundle: Crea un archivo que contiene un repositorio completo o parte de él.
    • git cherry: Muestra confirmaciones que están en una rama pero no en otra.

GitHub

GitHub es una plataforma de desarrollo colaborativo que utiliza Git como sistema de control de versiones. Permite a los desarrolladores alojar sus proyectos, colaborar con otros y compartir su código de manera eficiente. GitHub ofrece características como seguimiento de problemas, revisiones de código y gestión de proyectos, lo que lo convierte en una herramienta esencial para equipos de desarrollo y comunidades de código abierto.

  • Crear proyecto con rama desde GitHub
    • Crear un nuevo repositorio en GitHub.
    • Seleccionar la opción "Initialize this repository with a README".
    • Seleccionar la opción "Add .gitignore" y elegir el tipo de proyecto.
    • Seleccionar la opción "Add a license" y elegir la licencia adecuada.
    • Hacer clic en "Create repository".
    • Clonar el repositorio en tu máquina local usando "git clone".
    • Crear una nueva rama usando "git checkout -b nombre-de-la-rama".
    • Realizar cambios en el proyecto y confirmar los cambios usando "git add" y "git commit".
    • Subir los cambios a GitHub usando "git push origin nombre-de-la-rama".
    • Crear un Pull Request en GitHub para fusionar los cambios de la rama en la rama principal.
    • Revisar y aprobar el Pull Request.
    • Fusionar el Pull Request en la rama principal.
    • Eliminar la rama en GitHub si ya no es necesaria.
